# `/posts` 页面与全站布局视觉改造设计

## 目标

将当前 Astro 站点的 `/posts` 页面及共享布局调整为接近 [baoyu.io](https://baoyu.io/) 的阅读体验。重点不是逐像素复刻，而是迁移其核心视觉语言：窄内容列、明显留白、克制的层级、正文优先的文章列表，以及轻量的链接交互。

本次改造覆盖：

- `src/layouts/Layout.astro`
- `src/layouts/ArticleLayout.astro`
- `src/pages/posts/index.astro`
- 与这些布局直接相关的页面适配，例如 `src/pages/index.astro`

## 当前状态

- `Layout.astro` 只提供最基础的 HTML 壳和全局字体设置，没有统一的内容容器、页头或阅读节奏。
- `/posts` 目前是单行列表，结构为“标题 - 描述”，信息密度高，但阅读感较弱。
- `ArticleLayout.astro` 使用较重的边框、标签胶囊和代码/引用块装饰，整体更像通用模板，不像极简博客。
- `index.astro` 只是跳转页，当前视觉上与目标风格不一致。

## 设计原则

1. 让页面首先像“可持续阅读的博客”，其次才像“站点模板”。
2. 通过排版和留白建立层次，少依赖边框、背景块和显眼装饰。
3. 让文章列表本身拥有正文感，而不是目录感。
4. 保持实现克制，不额外引入目标站没有必要的数据结构或交互能力。

## 方案选择

采用“轻量高还原”方案：

- 对齐 `baoyu.io` 的版式气质和阅读节奏。
- 保留当前站点的数据来源和路由结构。
- 不实现分页、主题切换、分类频道等超出当前需求的能力。

## 页面与组件设计

### 1. 全站 `Layout`

将 `Layout.astro` 提升为统一页面骨架。

计划包含：

- 一个居中的窄主列容器，桌面端控制在适合阅读的宽度范围内。
- 极简页头，至少提供站点名和主要入口链接，避免空白页面缺少方向感。
- 统一全局颜色、字体、链接和基础元素样式。
- 通过全局样式为标题、段落、列表、时间信息提供一致的默认节奏。

视觉特征：

- 亮色背景，接近白纸感。
- 深灰正文，不使用高饱和主色作为大面积视觉中心。
- 链接默认低干扰，悬停时再增强识别。

### 2. `/posts` 列表页

将当前单行列表改为文章流式排版。

每篇文章块包含：

- 标题：作为主要点击入口，字号明显高于摘要。
- 摘要：单独一段显示，承担内容预览。
- 日期：弱化呈现，作为辅助信息。
- 操作文案：可保留轻量入口文案，例如“阅读全文”或简洁箭头式链接，但不会做成按钮。

页面标题将不再使用当前“首页”这种与路由不匹配的命名。默认改为更明确的博客列表标题，例如“文章”或“Posts”。如果实现时发现现有站点文案有更清晰的约定，则遵循站点上下文。

列表布局要求：

- 相邻文章块之间依靠垂直间距分隔，不使用卡片。
- 移除当前连字符式标题/描述串联结构。
- 保持移动端一列阅读，不做复杂响应式重排。

### 3. 文章正文页 `ArticleLayout`

将正文页对齐到同一套阅读语言。

调整方向：

- 标题更克制，减少模板化“大标题 + 粗边框”的感觉。
- 元信息改为轻量文本行，保留作者与日期，但弱化视觉权重。
- 标签从胶囊式装饰改为更轻的文本式或简洁行内样式。
- 正文段落行高、标题间距、列表节奏更像长文阅读页面。
- 引用、代码块、行内代码保留可读性，但去掉过强的装饰感。

### 4. 首页跳转页 `index.astro`

由于首页当前会跳转到 `/posts`，它只需要与新的全站布局保持一致，不必单独设计为营销页或落地页。

如果最终实现中跳转内容可完全隐藏且不影响使用，可以保留最小可见内容；如果保留提示文案，则让其视觉风格和新布局一致。

## 数据与实现方式

- 继续使用 `import.meta.glob('./*.md', { eager: true })` 获取文章列表，不引入集合重构。
- 在 `/posts` 页面内完成排序与渲染结构调整。
- 优先通过已有 Astro 单文件组件的局部 `<style>` 完成样式变更。
- 如果多个页面共享同一组基础样式，则上提到 `Layout.astro` 的全局样式中。

## 错误处理与兼容性

- 缺少 `description` 的文章仍需安全渲染，允许摘要区域回退为空或隐藏。
- 缺少 `pubDate` 时，排序逻辑保持当前行为或做安全回退，但不在本次扩展复杂的日期校验系统。
- 移动端需保证标题、摘要、元信息不重叠，不依赖悬停才能完成核心阅读。

## 测试与验证

在实现完成后，至少执行以下验证：

1. 运行 `pnpm build`，确保 Astro 构建通过。
2. 启动本地开发服务器，检查 `/posts`、首页跳转页和一篇文章详情页。
3. 在桌面和移动宽度下确认：
   - 主列宽度和留白合理。
   - 文章标题、摘要、日期层级清晰。
   - 正文页标题、引用、代码块和列表可读。
   - 页面没有出现卡片化、按钮化或过度装饰的退化。

## 非目标

本次不包含：

- 新增分页系统
- 新增主题切换
- 新增博客分类、翻译频道或 RSS 入口重构
- 重写文章数据模型
- 逐像素复刻 `baoyu.io` 的完整信息架构

## 实施顺序

1. 先调整 `Layout.astro`，建立统一骨架和全局排版基线。
2. 重构 `/posts` 列表结构与样式。
3. 收敛 `ArticleLayout.astro`，让详情页与列表页一致。
4. 处理 `index.astro` 的最小适配。
5. 运行构建并进行本地页面检查。
